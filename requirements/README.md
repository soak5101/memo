# 要件情報フォルダ

このフォルダは、プロジェクトの要件に関するドキュメントを管理するための場所です。

## フォルダ構成

- **機能要件/**: システムが実現すべき機能に関する要件
- **非機能要件/**: パフォーマンス、セキュリティ、使いやすさなどに関する要件
- **ユースケース/**: システムの使用シナリオ
- **設計書/**: 要件に基づいた設計ドキュメント

## ドキュメント形式

要件ドキュメントは以下の形式で作成することを推奨します：
- Markdown (.md)
- PDF
- Word文書 (.docx)
- 図表 (PNG, JPG, SVG)

## 命名規則

ファイル名は以下の規則に従ってください：
- 日付を含める場合: `YYYY-MM-DD_ドキュメント名.拡張子`
- バージョンを含める場合: `ドキュメント名_v1.0.拡張子`
